## Story
**As a** paciente autenticado,
**I want** falar diretamente com o concierge clicando em um botão de voz,
**so that** possa obter orientações de saúde sem precisar digitar, facilitando o uso para o público idoso.

## Acceptance Criteria

1. Um botão de microfone está visível e acessível na interface de chat (ao lado do campo de texto)
2. Ao clicar no botão, a gravação de áudio inicia (feedback visual: botão muda de cor/ícone para "gravando")
3. Ao clicar novamente, a gravação para e o áudio é automaticamente processado e enviado
4. O áudio capturado é convertido para formato `.wav` antes do envio
5. O frontend chama `POST /api/atendimento/upload-interacao` enviando o arquivo `.wav` como `multipart/form-data` com os campos: `file`, `Prompt` (vazio), `JsonHistory`, `PatientId`, `PatientName`
6. A mensagem do usuário aparece no histórico como "[Mensagem de voz]" enquanto aguarda a resposta
7. A resposta do backend é renderizada como mensagem do concierge no histórico
8. Em caso de erro (API, permissão de microfone negada, browser incompatível), uma mensagem amigável é exibida
9. O botão de voz fica desabilitado durante o processamento da requisição

## ⚠️ Pré-condição crítica (blocker)
**Esta story depende de confirmação do backend.** O endpoint `upload-interacao` usa `[FromBody]` e `[FromForm]` simultaneamente, o que é inválido em ASP.NET Core. Antes de implementar, confirmar com o time de backend que a correção foi aplicada (`[FromForm] InteractRequest request`). Ver `docs/architecture.md` — seção "Risco Crítico".

## Dev Notes

### `src/hooks/useVoiceRecorder.ts`

```typescript
import { useRef, useState } from 'react';

export function useVoiceRecorder() {
  const [isRecording, setIsRecording] = useState(false);
  const mediaRecorderRef = useRef<MediaRecorder | null>(null);
  const chunksRef = useRef<BlobPart[]>([]);

  const startRecording = async (): Promise<void> => {
    const stream = await navigator.mediaDevices.getUserMedia({ audio: true });
    const recorder = new MediaRecorder(stream);
    chunksRef.current = [];

    recorder.ondataavailable = e => {
      if (e.data.size > 0) chunksRef.current.push(e.data);
    };

    mediaRecorderRef.current = recorder;
    recorder.start();
    setIsRecording(true);
  };

  const stopRecording = (): Promise<File> => {
    return new Promise((resolve, reject) => {
      const recorder = mediaRecorderRef.current;
      if (!recorder) { reject(new Error('No recorder')); return; }

      recorder.onstop = async () => {
        const blob = new Blob(chunksRef.current, { type: recorder.mimeType });
        const wavFile = await convertToWav(blob);
        recorder.stream.getTracks().forEach(t => t.stop());
        resolve(wavFile);
      };

      recorder.stop();
      setIsRecording(false);
    });
  };

  return { isRecording, startRecording, stopRecording };
}
```

### Conversão para WAV

O `MediaRecorder` no Chrome grava em `audio/webm` por padrão. É necessário converter para `.wav`:

**Opção A — Biblioteca (recomendada para hackathon):**
```
npm install audiobuffer-to-wav
```

```typescript
import audioBufferToWav from 'audiobuffer-to-wav';

async function convertToWav(blob: Blob): Promise<File> {
  const audioCtx = new AudioContext();
  const arrayBuffer = await blob.arrayBuffer();
  const audioBuffer = await audioCtx.decodeAudioData(arrayBuffer);
  const wavBuffer = audioBufferToWav(audioBuffer);
  return new File([wavBuffer], 'audio.wav', { type: 'audio/wav' });
}
```

**Opção B — Verificar suporte nativo:**
```typescript
// Alguns browsers suportam audio/wav nativamente
const mimeType = MediaRecorder.isTypeSupported('audio/wav') ? 'audio/wav' : 'audio/webm';
```

### Contrato da API — Endpoint de voz
- **URL:** `POST /api/atendimento/upload-interacao`
- **Content-Type:** `multipart/form-data` (NÃO setar manualmente — o browser seta com boundary)
- **Campos:**
  - `file`: File object com nome `audio.wav` e type `audio/wav`
  - `Prompt`: string vazia `""`
  - `JsonHistory`: `JSON.stringify(history.map(m => ({role, content})))`
  - `PatientId`: string
  - `PatientName`: string
- **⚠️ NÃO enviar `Content-Type: application/json`** — usar `FormData` diretamente

### Função `sendVoice` no `useChatHistory`

```typescript
const sendVoice = async (wavFile: File, session: PatientSession) => {
  const userMsg: Message = {
    role: 'user',
    content: '[Mensagem de voz]',
    timestamp: Date.now(),
    type: 'voice',
  };
  const updatedMessages = [...messages, userMsg];
  setMessages(updatedMessages);
  setIsLoading(true);

  try {
    const responseText = await sendVoiceMessage(wavFile, updatedMessages, session);
    const assistantMsg: Message = {
      role: 'assistant',
      content: responseText,
      timestamp: Date.now(),
      type: 'text',
    };
    setMessages(prev => [...prev, assistantMsg]);
  } catch {
    setMessages(prev => [...prev, {
      role: 'assistant',
      content: 'Não foi possível processar sua mensagem de voz. Tente novamente.',
      timestamp: Date.now(),
      type: 'text',
    }]);
  } finally {
    setIsLoading(false);
  }
};
```
